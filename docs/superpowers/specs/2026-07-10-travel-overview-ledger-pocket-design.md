# 旅行概览与索引夹层优化设计

## 目标

在保持现有皮革旅行档案视觉体系和静态站点架构不变的前提下，重构“旅行概览”和“索引夹层”：消除重复按钮、重复选项与重复状态说明，让索引夹层成为唯一的高级筛选中心，并让概览与筛选结果提供更丰富、可信的数据统计。

## 核心原则

- 索引夹层是高级筛选与排序的唯一入口。
- 左页只承担搜索、年份快捷筛选、结果摘要和旅行记录浏览。
- 删除独立路线筛选抽屉及其所有入口，不保留“打开筛选面板”按钮。
- 筛选条件变化后立即更新列表、URL 和动态统计，不设置“应用筛选”按钮。
- 相同状态只展示一次：控件本身已经表达当前值时，不再重复显示“当前筛选”或“当前排序”。
- 仅使用现有旅行数据能够可靠推导的指标，不虚构距离、费用、天数等信息。

## 信息架构

### 路线档案左页

从上到下保留：

1. 页面标题。
2. 全文搜索框。
3. 年份快捷书签。
4. 结果数量说明。
5. 分组后的旅行记录列表。

左页删除“筛选与排序”按钮。搜索框和年份书签继续与高级筛选共享同一份路由状态。

### 索引夹层右页

从上到下组织为：

1. 动态结果快照：命中记录、覆盖城市、覆盖省份和日期跨度。
2. 时间与地点：年份、月份、省份、城市。
3. 记录特征：首次/再次到访、照片状态。
4. 排序方式：最新、最早、地点、省份、标题。
5. 唯一操作“重置全部”。

省份和城市联动；选择省份后，城市列表只显示该省份的城市。所有控件即时生效。

### 旅行概览右页

旅行概览专注全量旅行数据洞察，不承担筛选控制。保留现有记录跨度、国家、省份、城市、年度高峰、常出发月份、高频省份和复访地点，同时补充：

- 复访率：再次到访记录数占全部记录数的比例。
- 活跃年份：存在旅行记录的年份数量。
- 活跃月份：一年十二个月中曾出现旅行记录的月份数量。
- 最长记录间隔：相邻两条旅行记录之间的最大天数，并显示对应日期区间。

统计卡片使用现有开放式纸张分区和紧凑卡片样式，不新增独立按钮。可以自然跳转到已有列表或地点页面的指标保留链接能力；纯统计指标保持非交互，避免制造伪入口。

## 组件与职责

### 数据派生

`deriveTravelModel` 负责生成全量概览统计，包括复访率、活跃年份、活跃月份和最长记录间隔。数据派生函数保持纯函数风格，输入旅行记录，输出结构化统计。

`getLedgerSnapshot` 负责根据当前筛选结果计算动态快照。零结果时返回计数为零、日期跨度为“暂无匹配记录”的稳定结构。

### 筛选控件

索引夹层复用现有 `data-ledger-filter` 和 `data-ledger-toggle` 事件契约，避免增加新的状态层。下拉框和分段控件均通过当前 hash 参数渲染选中状态。

“重置全部”恢复 `LEDGER_FILTER_DEFAULTS`，包括最新优先排序；默认状态下按钮禁用。

### 路由状态

所有筛选、搜索和排序继续序列化到 `#ledger` 的查询参数中，保证刷新、前进后退和复制链接后状态一致。条件变化使用 `history.replaceState`，避免每次选择都产生一条浏览器历史记录。

## 删除范围

- 删除 `index.html` 中的顶部筛选按钮。
- 删除 `journal-drawer` 容器。
- 删除抽屉渲染、打开、关闭、焦点恢复及相关键盘处理逻辑。
- 删除索引夹层的快捷排序按钮、“打开筛选面板”按钮、“当前筛选”重复摘要和“当前排序”重复说明。
- 删除左页“筛选与排序”按钮。
- 删除只服务于上述元素的 CSS；保留仍被其他页面使用的通用样式。

## 响应式行为

- 桌面端维持左右纸页布局：左页列表，右页筛选工作台。
- 窄屏单列布局中，索引夹层排在路线列表之前，使用户先设置条件再浏览结果。
- 时间/地点下拉框在可用宽度允许时两列排列，较窄宽度下切换为单列。
- 分段控件允许等宽收缩，不产生横向滚动。
- 统计文本允许安全换行，数字采用现有等宽字体和数字对齐风格。

## 空状态与容错

- 筛选结果为空时，左页显示“没有找到匹配的旅行记录”，右页计数归零且日期跨度显示“暂无匹配记录”。
- 数据缺少日期时忽略该记录的日期间隔计算；没有足够日期时，最长间隔显示“暂无足够记录”。
- 数据缺少照片数组时按无照片处理，保持现有筛选语义。
- 删除抽屉后，键盘 Esc 仅处理日记详情等仍存在的浮层，不执行无效的抽屉关闭逻辑。

## 验证标准

### 功能

- 年份、月份、省份、城市、到访类型、照片状态和排序均能即时更新左页。
- 省份变化后城市选项正确联动，失效的城市条件自动归一化。
- 搜索框、年份书签和右页控件组合筛选正确。
- URL 参数与控件状态一致，刷新及前进后退后可恢复。
- 重置全部恢复默认结果并在默认状态禁用。
- 页面中不存在路线筛选抽屉及其打开入口。

### 数据

- 全量统计与 `data/travel_data.json` 一致。
- 动态统计基于筛选后的记录集合计算。
- 复访率、活跃年份、活跃月份和最长记录间隔对空数据及缺失日期安全。

### 视觉与可访问性

- 桌面端左右页无内容遮挡或意外溢出。
- 窄屏下高级筛选先于列表出现且无横向滚动。
- 所有表单控件有可感知标签，分段按钮通过 `aria-pressed` 表达状态。
- 结果数量变化通过现有 `aria-live` 区域告知。
- 交互控件保留清晰的键盘焦点样式。

## 实现约束

- 不引入新依赖、框架或构建步骤。
- 继续使用现有原生 JavaScript、hash 路由和 CSS 架构。
- 保留工作区中已有的 `js/app.js` 与 `css/journal.css` 未提交优化，并在其基础上增量修改。
